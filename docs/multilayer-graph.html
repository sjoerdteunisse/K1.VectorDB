Multilayer Graph

How data builds up
layer by layer

K1.VectorDB organises knowledge into named layers. Nodes live on one or more layers, edges connect them within a layer or across layers, and every layer gets its own vector index for semantic search.

Data Flow

From adding a node to retrieving it by semantic search — five stages, two storage targets.

🗂️

1. AddLayer

Create a named layer. A dedicated VectorDb index is registered for it.

✏️

2. AddNode

A GraphNode is created with Label, Content, LayerIds and Metadata.

🧠

3. Embed

Content is sent to IEmbedder (LM Studio) and a float[] vector is returned.

📐

4. Index

The vector is inserted into the layer's HNSW index inside VectorDb.

🔍

5. Search

A query is embedded and cosine distances computed across the HNSW index.

↩️

6. Traverse

Top-K results seed BFS; edges carry the walk across layers.

Edge Types

Edges are classified at query time — not stored as a type flag — based on whether source and target share a common layer.

Type Visual Rule API method Use case
Intra-layer ──► Source and target share at least one LayerId GetIntraLayerEdges(layerId) Relationships within a single semantic domain
Inter-layer - - ► Source and target share no LayerId GetInterLayerEdges() Bridges between domains, e.g. concept → implementation
Directed IsDirected = true A → B only; GetInEdges / GetOutEdges differ AddEdge(..., directed: true) Causal, hierarchical, or implementation relationships
Undirected IsDirected = false A ↔ B; traversal follows both directions AddEdge(..., directed: false) Symmetric associations — "related-to", "similar-to"

A node's LayerIds list can contain more than one Id — this is how a shared node appears in multiple layers simultaneously (see the ghost "Graph Theory" circle in Layer 1 above). When a shared node's edge target is also in one of those same layers, the edge is still intra-layer.

Code Example

Building the three-layer graph shown in this proposal — from scratch in C#.

// ── Bootstrap ────────────────────────────────────────────────────
var embedder = new LMStudioEmbedder { Model = "Qwen/Qwen3-Embedding-0.6B-GGUF" };
var graph    = new MultiLayerGraph(embedder, "data/graph");

// ── Layers ───────────────────────────────────────────────────────
var domain  = graph.AddLayer("Domain",     "High-level domain concepts");
var tech    = graph.AddLayer("Technology", "Concrete tools and algorithms");
var docLayer= graph.AddLayer("Document",   "Raw text documents and chunks");

// ── Domain nodes ─────────────────────────────────────────────────
var graphTheory  = graph.AddNode(domain.Id,  "Graph Theory",
    "Mathematical study of graphs — vertices connected by edges.");
var vectorSearch = graph.AddNode(domain.Id,  "Vector Search",
    "Semantic retrieval using embedding vectors and cosine similarity.");

// ── Technology nodes ─────────────────────────────────────────────
var hnsw      = graph.AddNode(tech.Id, "HNSW Index",
    "Hierarchical Navigable Small World graph for ANN search.");
var lmStudio  = graph.AddNode(tech.Id, "LM Studio Embedder",
    "Local LLM inference server providing text embeddings via HTTP.");

// ── Document nodes ───────────────────────────────────────────────
var docIntro  = graph.AddNode(docLayer.Id, "Doc: Intro to Graphs",
    "An introductory article explaining graph theory concepts.");
var docVec    = graph.AddNode(docLayer.Id, "Doc: Vector Similarity",
    "A technical document covering cosine similarity and ANN algorithms.");

// ── Shared node: Graph Theory also lives in Technology layer ─────
graph.PlaceNodeInLayer(graphTheory.Id, tech.Id);

// ── Intra-layer edges ────────────────────────────────────────────
graph.AddEdge(graphTheory.Id,  vectorSearch.Id, "enables",   weight: 0.95);
graph.AddEdge(hnsw.Id,         lmStudio.Id,    "uses",      weight: 1.0);
graph.AddEdge(docIntro.Id,     docVec.Id,      "references",weight: 0.9);

// ── Inter-layer edges (domain → technology) ──────────────────────
graph.AddEdge(graphTheory.Id,  hnsw.Id,     "implemented-by", weight: 0.85);
graph.AddEdge(vectorSearch.Id, lmStudio.Id, "powered-by",     weight: 0.9);

// ── Inter-layer edges (technology → document) ────────────────────
graph.AddEdge(hnsw.Id,     docIntro.Id, "documented-in", weight: 0.75);
graph.AddEdge(lmStudio.Id, docVec.Id,   "documented-in", weight: 0.8);

// ── Semantic search ───────────────────────────────────────────────
var results = graph.SearchNodes("how does vector similarity work in practice?", topK: 5);

foreach (var r in results)
    Console.WriteLine($"[{r.Score:F2}] {r.Node.Label} (layers: {string.Join(", ", r.Node.LayerIds)})");

// ── Traversal from a search result ───────────────────────────────
var neighbours = graph.GetNeighbors(results[0].Node.Id, includeInterLayer: true);
var bfsWalk    = graph.BreadthFirstSearch(results[0].Node.Id, maxDepth: 3);

// ── Persist ───────────────────────────────────────────────────────
graph.Save();
